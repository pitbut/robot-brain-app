# Приложение "Робот — телефон-мозг" — сборка через GitHub Actions

Проект теперь полностью самодостаточный (корневые `build.gradle`/`settings.gradle`,
иконки, workflow) — GitHub соберёт APK сам, Android Studio локально не нужна.

## Шаг 1. Залить проект на GitHub

1. Распакуй `RobotPhoneApp.zip` — сохраняй структуру папок как есть.
2. github.com → **New repository** → назови, например, `robot-brain-app`.
3. Залей **всё содержимое** папки `RobotPhoneApp` (включая скрытую папку `.github/` —
   в веб-интерфейсе GitHub при перетаскивании папки она подхватится автоматически;
   если заливаешь через drag-n-drop файл за файлом, не забудь про `.github/workflows/build-apk.yml`).

Если через git:
```bash
cd RobotPhoneApp
git init
git add .
git commit -m "robot brain app"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЮЗЕРНЕЙМ/robot-brain-app.git
git push -u origin main
```

## Шаг 2. Дождаться сборки

1. На странице репозитория открой вкладку **Actions**.
2. Увидишь запущенный workflow **"Build APK"** — он стартует автоматически при пуше в `main`.
3. Сборка идёт 2-4 минуты (скачивает Gradle, Android SDK-компоненты, зависимости).
4. Когда появится зелёная галочка — открой этот workflow run.

## Шаг 3. Скачать APK

Внизу страницы запуска — раздел **Artifacts**, там будет `robot-brain-debug-apk.zip`.
Скачай, распакуй — внутри `app-debug.apk`. Это и есть готовое приложение.

## Шаг 4. Установить на телефон

1. Перекинь `app-debug.apk` на телефон (по кабелю, через Telegram себе, Google Drive — как угобно).
2. Открой файл на телефоне — Android спросит разрешение "Установить из неизвестных источников"
   (нормально для APK не из Google Play) — разреши один раз.
3. Установится обычное приложение "Robot Brain".

## Как пользоваться

1. Открой приложение на телефоне, закреплённом на роботе.
2. В поле сверху впиши домен сервера — без `https://`, просто
   `robot-platform-xxxx.onrender.com`.
3. Жми **"Начать трансляцию"** → разреши камеру, геолокацию, уведомления.
4. Если на телефоне нет Google Play Services for AR — приложение само
   предложит установить/обновить их (стандартный системный диалог ARCore).
   Без них видео и GPS всё равно будут работать, просто объезда препятствий
   по камере не будет — статус на экране это покажет.
5. Работает в фоне (foreground-сервис + уведомление в шторке), экран может блокироваться —
   трансляция не прервётся.
6. **"Остановить"** — кнопка в приложении или прямо в уведомлении в шторке.
   Закрывает сокет, GPS, компас, камеру полностью, без похода в настройки телефона.

## Как теперь распределены роли (важное отличие от первой версии)

- **Телефон = мозг**: сам знает GPS+компас, сам видит препятствия через
  камеру (ARCore Depth — карта глубины сцены без LiDAR) и сам решает,
  с какой скоростью крутить каждое колесо.
- **ESP32 = просто руки**: получает готовые команды на моторы (`phone_drive`)
  и крутит их. Сонары на плате теперь не обязательны — если когда-нибудь
  подключишь их обратно, их показания уйдут оператору просто для информации,
  решение всё равно остаётся за телефоном.
- Квадратик 120x120 на экране приложения — не настоящее превью камеры
  (это отдельный OpenGL-контекст, нужный ARCore для расчёта глубины),
  специально пустой. Реальное видео с камеры оператор видит на карте —
  собственному экрану телефона показывать картинку незачем (он висит на
  роботе, на его дисплей никто не смотрит).

## Если Actions выдаёт ошибку сборки

- Пришли скриншот вкладки Actions с красным крестиком — открою лог конкретного
  шага, который упал, и поправлю.
- Частая причина — если при заливке на GitHub случайно пропустилась папка
  `.github/workflows/` (её иногда не видно в проводнике, т.к. начинается с точки) —
  тогда вкладка Actions вообще не покажет workflow. Проверь, что файл
  `.github/workflows/build-apk.yml` есть в репозитории.

## Дальнейшая сборка при изменениях

Каждый `git push` в ветку `main` (или заливка новых файлов через веб-интерфейс)
автоматически запускает новую сборку — свежий APK всегда будет ждать во вкладке
Actions → последний run → Artifacts.
